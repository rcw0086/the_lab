## ADR-003: API Style (REST First)

**Decision**  
Use RESTful APIs with consistent error semantics.

**Rationale**

- Simple and debuggable
- Excellent FastAPI + TS fit

**Consequences**

- Some chatty interactions
- Eventing can be added later
